## 1. `flex-grow: 1` + `flex-basis: 0`, 왜 같이 쓸까?

로그인 카드 UI에서 이런 코드를 마주쳤다.

```css
.signin__card .actions a {
  flex-grow: 1;
  flex-basis: 0;
  text-align: center;
  padding: 20px;
  color: #555;
  font-size: 14px;
  border-right: 1px solid #ddd;
}
```

"로그인 / 회원가입" 같은 탭 버튼을 나란히 배치하는 구조인데, `flex-basis: 0`이 눈에 걸려서 오늘은 이 속성을 제대로 짚고 넘어가기로 했다.

### `flex-basis`란

`flex-grow`와 `flex-shrink`가 여유 공간을 분배하기 **전의 기준(시작) 크기**를 정하는 속성.

- `flex-basis: auto` (기본값): 요소의 `width` 또는 콘텐츠 크기를 기준으로 삼음
- `flex-basis: 0`: 콘텐츠 크기를 무시하고 기준 크기를 0으로 취급

### `flex-grow: 1` + `flex-basis: 0`의 조합 의미

이 둘을 같이 쓰면 "**콘텐츠 길이와 무관하게 모든 아이템을 완전히 동일한 너비로 나눠라**"는 의미가 됨.

동작 순서:

1. 모든 아이템의 기준 크기를 0으로 만듦 → 텍스트가 길든 짧든 초기 크기는 전부 0
2. 컨테이너의 여유 공간을 `flex-grow: 1`끼리 1:1로 균등 분배
3. 결과적으로 "로그인", "회원가입" 처럼 글자 수가 달라도 버튼 너비가 정확히 같아짐

### 만약 `flex-basis: auto`였다면?

```css
flex-grow: 1;
flex-basis: auto;
```

이 경우엔 먼저 각 아이템이 자기 콘텐츠 크기만큼 공간을 차지하고, 남는 여유 공간만 `flex-grow` 비율로 나눠 갖는다. 그러면 텍스트가 긴 아이템은 여전히 더 넓게, 짧은 아이템은 더 좁게 남을 수 있음. 즉 **완전한 균등 너비를 보장하지 않는다.**

### Shorthand로 줄이면

```css
.signin__card .actions a {
  flex: 1; /* flex-grow: 1; flex-shrink: 1; flex-basis: 0%; */
}
```

`flex: 1`은 `flex-basis`가 자동으로 `0%`가 되기 때문에 원본 코드(`flex-grow: 1; flex-basis: 0;`)와 사실상 같은 결과를 낸다.

| 표기                           | flex-grow | flex-shrink | flex-basis |
| ------------------------------ | --------- | ----------- | ---------- |
| `flex-grow: 1; flex-basis: 0;` | 1         | 1 (기본값)  | 0          |
| `flex: 1`                      | 1         | 1           | 0%         |

이 케이스에서는 둘 다 동일하게 동작한다.

------

## 2. 오늘의 정리

- 탭/버튼 그룹처럼 **아이템 너비를 완전히 균등하게** 맞추고 싶을 때는 `flex-grow: 1` + `flex-basis: 0` (또는 `flex: 1`) 조합을 쓴다.
- `border-right`로 구분선을 넣는 탭 UI라면, 마지막 아이템의 오른쪽 테두리를 `:last-child`로 제거하는 처리도 잊지 말 것.

```css
.signin__card .actions a:last-child {
  border-right: none;
}
```
