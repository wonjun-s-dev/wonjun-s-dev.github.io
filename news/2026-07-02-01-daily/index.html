1. position: fixed 기준점

position: fixed는 뷰포트(브라우저 화면) 기준으로 위치가 잡힌다.

핵심 특징

  • 부모 요소가 무엇이든 상관없이, 현재 보이는 화면(viewport)의 좌상단을 기준으로 top, left, right, bottom 값이 적용됨.

  • 스크롤을 해도 화면에 고정되어 같은 자리에 머묾. (예: 화면 우측 하단 “맨 위로 가기” 버튼)

  • 문서 흐름(document flow)에서 완전히 빠져나가서, 다른 요소들은 이 요소가 없는 것처럼 배치됨.

.scroll-top-btn {
  position: fixed;
  bottom: 20px;
  right: 20px;
}

→ 스크롤을 아무리 내려도 항상 화면 우측 하단 20px 지점에 고정.

주의할 예외 상황

부모(또는 조상) 요소에 다음 중 하나라도 걸려있으면, 뷰포트가 아니라 그 조상 요소 기준으로 바뀜:

  • transform
  • filter
  • perspective
  • will-change: transform 등
.parent {
  transform: translateX(0); /* 이게 있으면 */
}
.child {
  position: fixed; /* 뷰포트가 아니라 .parent 기준이 됨 */
}

라이브러리(Swiper, 모달 등) 쓸 때 position: fixed가 이상하게 동작하면, 보통 이 transform 때문인 경우가 많음.

2. 자식 선택자 vs 자손 선택자

header > .inner {
  height: 120px;
}

>는 직계 자식(direct child) 선택자. “하위요소(자손, descendant)”와는 다름.

→ header의 바로 한 단계 아래 자식인 .inner에만 적용됨.

비교 예시

<header>
  <div class="inner">          <!-- ✅ 적용됨: header의 직계 자식 -->
    <div class="inner">        <!-- ❌ 적용 안 됨: header의 자식이 아니라 손자 -->
    </div>
  </div>
</header>

선택자 종류 정리

선택자 의미
header .inner (공백) 자손 선택자 — 몇 단계든 상관없이 내부의 모든 .inner
header > .inner (>) 자식 선택자 — 바로 한 단계 아래만
header + .inner (+) 인접 형제 선택자 — 바로 다음 형제 하나
header ~ .inner (~) 일반 형제 선택자 — 이후의 모든 형제

실무에서는 스타일이 의도치 않게 중첩된 하위 요소까지 퍼지는 걸 막고 싶을 때 >를 자주 씀.

3. 수직 중앙 정렬 원리

header .logo {
  height: 75px;
  position: absolute;
  top: 0;
  bottom: 0;
  left: 0;
  margin: auto;
}

position: absolute 상태에서 top, bottom을 둘 다 지정하면, 요소의 높이가 정해져 있어도(height: 75px) 브라우저는 “이 요소가 위에서부터 0, 아래에서부터도 0인 영역 안에 있어야 한다”는 조건만 알 뿐, 그 안에서 어디에 배치할지는 알려주지 않은 상태가 됨.

margin이 없으면 왜 자동으로 중앙이 안 될까?

기본값인 margin: 0이 적용되면, 브라우저는 그냥 top: 0 기준으로 붙여버림.

header .logo {
  height: 75px;
  position: absolute;
  top: 0;      /* 위에서 0 */
  bottom: 0;   /* 아래에서 0 */
  left: 0;
  /* margin 없음 → margin: 0 0 0 0 (기본값) */
}

→ top: 0이 우선 적용되면서 그냥 맨 위에 붙어버림. bottom: 0은 무시되는 것처럼 보임 (정확히는 남는 공간이 margin 쪽으로 안 가고 그냥 버려지는 것).

margin: auto를 주면?

margin: auto는 “남는 공간을 똑같이 나눠서 채워라”라는 뜻. top: 0; bottom: 0;으로 이미 세로 방향의 전체 가용 공간(header의 높이만큼)이 정해졌고, 요소 자체의 높이는 75px로 고정되어 있으니:

남는 공간 = header 높이 - 75px
→ 이 남는 공간을 위/아래 margin에게 절반씩 자동 분배

그 결과 요소가 정확히 수직 중앙에 위치하게 됨.

정리

상황 결과
top: 0; bottom: 0; + margin 없음 top 기준으로 붙음 (위쪽 정렬)
top: 0; bottom: 0; + margin: auto 남는 공간을 위아래로 균등 분배 → 수직 중앙 정렬

참고로 left: 0만 있고 right는 없으니, 가로 방향은 margin: auto가 적용될 여지가 없어서 그냥 왼쪽에 붙음. 가로도 중앙 정렬하려면 left: 0; right: 0;을 같이 줘야 margin: auto가 가로에도 작동함.
