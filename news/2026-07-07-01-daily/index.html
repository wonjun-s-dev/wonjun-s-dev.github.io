1. top: calc(100% + 4px), 여기서 100%는 어디 기준?

드롭다운 메뉴 위치를 잡다가 이 코드를 봤다.

.dropdown {
  position: absolute;
  top: calc(100% + 4px);
}

top에서 쓰는 퍼센트는 자신의 containing block(기준이 되는 부모)의 height를 기준으로 계산된다.

축 기준 정리

  • top, bottom의 퍼센트 → 부모의 height 기준
  • left, right, width의 퍼센트 → 부모의 width 기준

가로/세로가 서로 다른 축 기준이라는 점이 헷갈리기 쉬운 부분이라 따로 기록해둔다.

top: calc(100% + 4px)의 의미

  • 100% = 부모(containing block) 높이만큼 아래로 내려간 지점 = 부모의 바로 아래(하단) 경계선
  • + 4px = 거기서 4px 더 내려간 위치

즉 “부모 바로 아래, 4px 간격을 두고” 라는 뜻이다.

실무 패턴 — 드롭다운/툴팁 배치

트리거 요소(버튼 등) 바로 아래에, 살짝 간격을 두고 드롭다운 메뉴를 배치할 때 자주 쓰는 패턴.

.dropdown-trigger {
  position: relative; /* dropdown-menu의 containing block이 됨 */
}
.dropdown-menu {
  position: absolute;
  top: calc(100% + 4px); /* 트리거 요소 바로 아래 + 4px 간격 */
  left: 0;
}

주의할 점

.dropdown-menu가 position: absolute일 때, top: 100%가 정확히 트리거 요소의 높이를 기준으로 계산되려면 가장 가까운 position: relative(또는 absolute/fixed) 조상이 바로 그 트리거 요소여야 한다. 더 상위 조상에 position: relative가 걸려있으면, 그 조상 기준으로 100%가 계산되어 의도한 위치에 안 붙는 문제가 생길 수 있다.

→ containing block 규칙은 position: fixed뿐 아니라 퍼센트 계산에도 그대로 적용된다는 걸 다시 한번 확인.
